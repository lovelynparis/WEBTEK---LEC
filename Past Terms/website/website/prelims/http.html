Hypertext Tranfer Protocol

Hypertext Tranfer Protocol or HTTP was one of the most widely used protocol in the world, it is used in the application layer that allows the communication and exchange of data and acts as the mediator between web browsers and servers.

HyperText Transfer Protocol (HTTP) was unsecure which was based on client-server architecture.It is TCP/IP based with a default port, TCP 80.

Version History:

HTTP/0.9 (1991)

The first version of HTTP, used in transferring raw data across the web.

HTTP/1.0 (Request For Comments (RFC) 1945, May 1996)

Improvement of HTTP/0.9 by allowing messages containing metainformation about the data being transferred in the MIME (Multipurpose Internet Mail Extensions) format.

https://tools.ietf.org/html/rfc1945

Terminologies used:

Client, aka User Agents (UA)

Web Browser, Web Crawlers/Spiders, Other End USer Tools and Applications

Server

Origin Server, Proxy Server (ex. Authentication Server), Gateways, Tunnels

Connection

Established for the purpose of communication between application programs

Message

Basic unit of HTTP communication, it is composed of textual information encoded in ASCII.

The web browser, server or proxy is usually the ones creating the messages using config files, APIs, and other interfaces.

Request

A message sent by the client to the server. This contains the first line of the message indicating the method to be applied, the resource, identifier and protocol version to be used.

HTTP Resource Addressing:

Identifies using Uniform Resource Identifer (URI; RFC 3986) or more specifically, HTTP URIs.

2 Subtypes of URI:

Uniform Resource Name (URN) - ex. International Standard Book Number (ISBN), Universal Product Code (UPC)

Uniform Resource Locator (URL) - ex. google.com

Authority

User information/authentication credentials (deprecated).

Host - domain name (resolved to an ip address using dns) of the server where the resource resides (or will be created).

Port Number

Path to Resource (resolved relative to the document root on the server)

Fragment Identifier - starts with a hashtag (#).

Protocol:

A set of rules that needs to be followed when messages are communicated in the Internet.

Examples:

HTTP(Hypertext Transfer Protcol) - a protocol used for tranferring messages over the Internet.

HTTPs - a protcol with a combination of HTTP and SSL or Secure Sockets Layer which makes the sharing of information more secure.

HTTP Response Message:

Status Line

The first line of the response message. It start with the protocol version and the numeric status code with the description.

Response Headers

This allows the server to pass additional messages that cannot be placed in the Status Line.

Message Body

The message body is used to carry the entity-body associated with the request or response. It only differs from the entity-body when transfer coding has been applied.

HTTP Request Methods:

GET

Request data from a specified source.

HEAD

Same as GET but returns only HTTP headers and no document body.

POST

Submit data to be processed to a specified resources.

OPTIONS

Represents a request for information about the communication options available on the request/response chain identified by the Request-URI.

TRACE

Used to invoke a remote, application-layer loop- back of the request message.

PUT

Uploads a representation of the specified URI.

DELETE

Deletes the specified resource.

CONNECT

Converts the request connection to a transparent TCP/IP tunnel.

Safe Methods:

GET, HEAD, OPTIONS, TRACE

Not Safe Methods:

POST, PUT, DELETE

Idempotent:

GET, HEAD, PUT, DELETE, OPTIONS, TRACE
